### 🏗️ 整体架构设计

#### 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                   用户浏览器访问                               │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│                 GitHub Pages 静态托管                          │
│  - index.html      前端网站                                   │
│  - index.json      OpenClaw 可发现的技能索引                   │
│  - 技能数据         每个技能一个 JSON 文件                      │
└────────────────────────────┬────────────────────────────────┘
                             │
             ┌───────────────┼───────────────┐
             ▼               ▼               ▼
    ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
    │  用户浏览     │  │OpenClaw 自动 │  │  技能安装     │
    │  技能发现     │  │ 发现技能      │  │  一键复制命令 │
    └──────────────┘  └──────────────┘  └──────────────┘
```

#### 架构特点

1. **全静态架构**：无需后端服务器，全部静态文件托管在 GitHub Pages
2. **自动发现**：遵循 OpenClaw SkillHub 索引格式，龙虾可以自动发现所有技能
3. **一键安装**：用户只需要复制安装命令，即可一键添加技能源和安装具体技能
4. **易于维护**：新增技能只需要在 `data/` 目录添加 JSON 文件，重新构建即可

### 📋 技能数据结构设计

每个技能遵循固定的 JSON 结构，只保留三个核心模块：

| 模块 | 字段 | 说明 | 必填 |
|------|------|------|------|
| **基本信息** | `name` | 技能名称 | ✅ |
| | `description` | 中文描述 | ✅ |
| | `descriptionEn` | 英文描述 | ⭕ |
| | `author` | 作者 | ✅ |
| | `repository` | Git 仓库地址 | ✅ |
| | `category` | 分类 | ✅ |
| | `tags` | 标签数组 | ✅ |
| | `license` | 开源协议 | ✅ |
| **Scene 场景描述** | `scene.problem` | 解决什么问题 | ✅ |
| | `scene.whoShouldUse` | 适合谁用 | ⭕ |
| | `scene.whenToUse` | 什么时候用（数组） | ⭕ |
| | `scene.whenNotToUse` | 什么时候不适用（数组） | ⭕ |
| **Showcase 效果展示** | `showcase.features` | 功能特点（数组） | ⭕ |
| | `showcase.result` | 最终效果说明 | ⭕ |
| **Usage 使用方式** | `usage.examples[].scene` | 示例场景 | ⭕ |
| | `usage.examples[].query` | 示例对话内容 | ⭕ |
| | `usage.tips[]` | 使用提示数组 | ⭕ |

**示例：**

```json
{
  "name": "optimize-website-style",
  "description": "帮你一键优化网站整体风格...",
  "scene": {
    "problem": "很多人自己做好网站后，界面很丑不知道怎么优化...",
    "whenToUse": [
      "网站做好了但看起来很丑，需要整体美化",
      "想更换主题颜色和设计风格"
    ]
  },
  "showcase": {
    "features": [
      "🔍 自动搜索当前主流网站设计风格",
      "🎨 给你展示多种风格选项"
    ],
    "result": "几分钟就能获得一个漂亮专业的网站界面"
  },
  "usage": {
    "examples": [
      {
        "scene": "优化现有网站风格",
        "query": "帮我优化网站风格"
      }
    ],
    "tips": [
      "✓ 说明你的问题",
      "✓ 说清楚你想要什么感觉"
    ]
  }
}
```

### 🎨 前端设计

#### 设计要点

1. **响应式布局**：使用 Tailwind CSS，支持移动端、桌面端
2. **卡片式设计**：每个技能一个独立卡片，清晰区分
3. **三色区分三大模块**：
   - 🎯 **场景描述**：紫色 `text-purple-700`
   - ✨ **效果展示**：蓝色 `text-blue-700`
   - 📖 **使用方式**：绿色 `text-green-700`
4. **一键复制**：所有代码块都配有复制按钮，点击一键复制到剪贴板
5. **底部引流**：页面底部放置微信二维码，引导定制开发咨询

#### 主要功能

| 功能 | 说明 |
|------|------|
| 一键添加源 | 首页提供添加技能源的命令，一键复制 |
| 技能列表 | 按卡片展示所有技能，清晰易读 |
| 分类标签 | 每个技能显示标签，便于浏览识别 |
| 三大模块展示 | 严格按照 scene + showcase + usage 展示 |
| 一键复制安装命令 | 每个技能都有一键复制安装按钮 |
| 响应式设计 | 手机上也能良好浏览 |

### 🔄 构建流程

1. **新增技能**：在 `data/` 目录创建 `skill-name.json`，按照标准结构填写
2. **构建索引**：运行 `npm run build`，脚本自动读取所有 JSON 生成 `index.json`
3. **部署**：推送到 GitHub，GitHub Pages 自动更新

```bash
# 完整构建部署流程
npm run build
git add .
git commit -m "add new skill: xxx"
git push
```

### 🚀 使用说明

#### 对于用户（添加技能源）

```bash
# 添加 SkillHub 源
npx skills source add https://Sxyman.github.io/skill-hub-website/index.json

# 搜索技能
npx skills find [关键词]

# 安装技能
npx skills add [技能名]
```

#### 对于站长（新增技能）

1. 在 `data/` 目录创建 `skill-name.json`
2. 按照标准结构填写三个模块
3. 运行 `npm run build` 重新生成索引
4. 提交到 Git 推送到 GitHub
5. GitHub Pages 自动更新
