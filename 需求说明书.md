# SkillHub - 精选 OpenClaw 技能导航需求说明书

## 项目背景

OpenClaw/ArkClaw 生态中，用户需要解决具体问题时，往往需要：
1. 自己一步步调教 AI，试错半小时
2. 不知道该用哪个技能，技能太多不知道怎么选
3. 零散技能不会组合使用
4. 很多技能发布后没人维护，找不到可用版本
5. 踩过的坑后人还要再踩一遍

SkillHub 要做的是：**精选实测技能导航站**，从全网收集技能，帮你筛选测试整理好，好用技能一站找齐，实测可用，降低你的使用门槛。所有技能全部免费，需要定制开发/安装指导/企业咨询时，扫码加微信获得专业服务。

---

## 核心模式

**All Free + WeChat引流**：
1. **所有技能全部免费** - 没有会员、没有付费技能、不需要token鉴权，零门槛使用
2. **帮你筛选测试** - 从全网收集技能，过滤掉不可用的，整理好给用户用
3. **微信引流变现** - 网站底部展示微信二维码，用户需要定制开发/安装指导/排障服务/企业咨询时，扫码加微信转化
4. **零成本起步** - 纯静态网站，GitHub Pages 免费托管，不需要服务器，不需要支付商户号

## 总体需求

### D01：网站展示
- ✅ 展示所有上架技能，清晰分类
- ✅ 每个技能卡片包含核心信息，点击可查看详情
- ✅ 响应式设计，手机和电脑都能正常浏览
- ✅ 顶部清晰说明网站定位：精选 OpenClaw 技能导航，好用技能一站找齐

### D02：OpenClaw 自动发现
- ✅ 输出符合 SkillHub 规范的 `index.json` 索引
- ✅ 用户添加源后，OpenClaw 能自动搜索发现技能
- ✅ 支持一键复制安装命令

### D03：一键复制功能
- ✅ 所有命令块右上角都有复制按钮
- ✅ 点击后显示"已复制"反馈
- ✅ 支持添加源命令、安装单个技能命令复制

### D04：中英双语描述
- ✅ 每个技能显示中文描述 + 英文描述
- ✅ 英文描述淡化显示，不抢视线

### D05：两种安装方式
- ✅ OpenClaw 一键安装命令（推荐）
- ✅ 手动 git 克隆方式（备选）

### D06：自动索引构建
- ✅ 从 `data/` 目录的 JSON 文件自动构建 `index.json`
- ✅ 支持批量构建

### D07：底部微信引流区
- ✅ 展示可提供的服务：定制开发、安装指导、排障服务、企业咨询
- ✅ 展示微信二维码，用户扫码直接添加
- ✅ 简洁大方，不影响主流程体验

### D20：极简技能内容结构（最终版）
每个技能**只保留三个核心模块**，删除其他所有模块。目的：让页面更简洁，用户一眼就能找到最关心的三个问题答案。

用户打开技能详情页，最关心的只有三个问题：
1. **❓ 这是什么？解决我什么问题？** → **场景描述**
2. **✨ 有什么效果？能帮我做到什么程度？** → **案例展示/效果说明**
3. **📝 我具体怎么用？给我个例子** → **使用方式**

删除所有其他模块：自动化工作流、FAQ、最佳实践等都去掉。让页面更干净。

## 技能 JSON 结构规范（极简版）

每个技能在 `data/` 目录下对应一个 JSON 文件，必须包含以下字段：

```json
{
  "name": "技能唯一标识",
  "description": "中文一句话描述",
  "descriptionEn": "英文一句话描述",
  "author": "作者名称",
  "repository": "Git 仓库地址",
  "homepage": "主页地址",
  "version": "版本号",
  "category": "分类（frontend/backend/tool/office/etc）",
  "verified": true,
  "stars": 0,
  "license": "MIT|proprietary",
  "tags": ["标签1", "标签2"],
  
  /* === D20 三个核心模块（只保留这三个） === */
  
  // 1️⃣ 场景描述：解决什么问题？谁该用？
  "scene": {
    "problem": "这个技能解决什么具体问题？一句话说清楚",
    "whoShouldUse": "什么样的人/在什么场景下应该用这个技能？",
    "whenToUse": ["什么时候用", "列几个典型场景（数组）"],
    "whenNotToUse": ["什么时候不推荐用（可选，帮助用户判断）"]
  },
  
  // 2️⃣ 案例展示/效果说明：能做到什么效果？有什么能力？
  "showcase": {
    "features": [
      "🔍 能力一：具体能做什么",
      "✨ 能力二：具体能做什么",
      "🛠️ 能力三：具体能做什么"
    ],
    "result": "最终能达到什么效果？一句话总结"
  },
  
  // 3️⃣ 使用方式：具体怎么用？给我例子
  "usage": {
    "examples": [
      {
        "scene": "场景名称",
        "query": "用户提问示例（可直接复制使用）"
      }
    ],
    "tips": [
      "✓ 提问技巧一：怎么说更容易得到正确结果",
      "✓ 提问技巧二：怎么说更容易得到正确结果"
    ]
  },
  
  "createdAt": "ISO 日期格式"
}
```

**重要：只保留 `scene` + `showcase` + `usage` 三个核心模块，删除所有其他模块（FAQ、workflow、bestPractice 等全部删除）。**
**删除说明：不再有 tier（分级）字段，所有技能全部免费。**
